# Skills Frameworks

Comprehensive skill maps by role and level.

---

## How to Use This Framework

1. **Identify target role and level** (e.g., Senior Engineer IC3)
2. **Review required skills** for that level
3. **Assess current proficiency** (1-5 scale)
4. **Identify gaps** (where current < required)
5. **Create development plan** to close gaps

---

## Software Engineer Framework

### IC1 (Junior Engineer)

**Technical Skills:**
- Basic programming in 1-2 languages (Level 2-3)
- Understands data structures and algorithms (Level 2)
- Can write tests (Level 2)
- Follows code review feedback (Level 2)
- Uses version control (Git) (Level 2)

**Core Skills:**
- Communication: Asks good questions (Level 2)
- Problem-solving: Can debug with guidance (Level 2)
- Learning: Actively learning new concepts (Level 3)

**Leadership:**
- N/A at this level

**Timeline to next level:** 12-24 months

---

### IC2 (Mid-Level Engineer)

**Technical Skills:**
- Proficient in primary language/stack (Level 3)
- Solid CS fundamentals (Level 3)
- Writes clean, maintainable code (Level 3)
- Writes comprehensive tests (Level 3)
- Participates meaningfully in code reviews (Level 3)
- Understands system architecture basics (Level 2-3)

**Core Skills:**
- Communication: Writes clear docs, asks for help (Level 3)
- Problem-solving: Debugs autonomously (Level 3)
- Project management: Estimates and delivers tasks (Level 3)

**Leadership:**
- Helps onboard new teammates (Level 2)

**Timeline to IC3:** 18-36 months

---

### IC3 (Senior Engineer)

**Technical Skills:**
- Expert in primary stack (Level 4)
- Strong systems design skills (Level 3-4)
- Writes production-quality code (Level 4)
- Drives technical decisions for features (Level 3)
- Understands performance and scalability (Level 3)
- Can work across the stack if needed (Level 3)

**Core Skills:**
- Communication: Influences team through writing and speaking (Level 4)
- Problem-solving: Breaks down complex problems (Level 4)
- Project management: Owns features end-to-end (Level 4)

**Leadership:**
- Mentors IC1-IC2 engineers (Level 3)
- Code review quality and feedback (Level 4)
- Influences team technical standards (Level 3)

**Timeline to IC4:** 24-48 months

---

### IC4 (Staff Engineer)

**Technical Skills:**
- Deep expertise in systems and architecture (Level 4-5)
- Drives technical strategy for systems/platforms (Level 4)
- Makes architecture decisions with broad impact (Level 4)
- Identifies and solves systemic technical problems (Level 4)

**Core Skills:**
- Communication: Writes influential technical docs (RFCs, ADRs) (Level 4)
- Problem-solving: Solves ambiguous, complex problems (Level 5)
- Strategic thinking: Thinks 12-24 months ahead (Level 4)

**Leadership:**
- Mentors across teams (Level 4)
- Drives technical direction for multiple teams (Level 4)
- Influences eng org technical culture (Level 4)

**Timeline to IC5:** 36-60+ months

---

### IC5 (Principal/Distinguished Engineer)

**Technical Skills:**
- Company-wide technical authority (Level 5)
- Drives multi-year technical vision (Level 5)
- Solves company-defining technical problems (Level 5)

**Core Skills:**
- Communication: Thought leadership (internal and external) (Level 5)
- Strategic thinking: Multi-year technical roadmap (Level 5)

**Leadership:**
- Shapes engineering culture company-wide (Level 5)
- External thought leadership (talks, writing, open source) (Level 5)
- Develops other technical leaders (Level 5)

## Skill Proficiency Definitions

### Technical Skills

**Level 1 (Beginner):**
- Learning fundamentals
- Needs significant guidance
- Can complete simple tasks with help

**Level 2 (Intermediate):**
- Understands concepts
- Can complete tasks with occasional help
- Improving through practice

**Level 3 (Proficient):**
- Autonomous on standard work
- Reliable and consistent
- Can teach basics to others

**Level 4 (Advanced):**
- Expert in area
- Handles complex scenarios
- Teaches and mentors others
- Drives improvements

**Level 5 (Master):**
- Company-wide authority
- Recognized externally
- Creates new approaches
- Shapes industry practices

## Using This Framework

1. **Identify current role/level and target**
2. **Review skills for target level**
3. **Assess current proficiency for each skill** (1-5)
4. **Calculate gaps** (target - current)
5. **Prioritize gaps** (critical vs. nice-to-have)
6. **Create development plan**

**Example:**
- Current: IC2 Engineer
- Target: IC3 Senior Engineer
- Gap: Systems design (Current: 2, Target: 4, Gap: +2)
- Priority: High (critical for IC3)
- Development: Architecture course + own system design + mentorship

Great skills frameworks provide clarity on expectations and paths for growth.
